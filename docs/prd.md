# 需求文档

## 1. 应用概述

### 1.1 应用名称
演艺排班管理工具

### 1.2 应用描述
为上海酒吧驻唱圈职业演艺中介提供的排班管理工具，用于管理多个酒吧的歌手和乐手演出排班，支持档期管理、智能排班辅助、顶班匹配、版本管理、记账结算等功能。应用以移动端为主，针对手机端操作场景进行优化设计。

## 2. 用户与使用场景

### 2.1 目标用户
演艺中介（单一用户，登录后管理所有数据）

### 2.2 核心使用场景
- 管理多个酒吧的歌手排班
- 处理顶班替换
- 薪酬结算核对

## 3. 页面结构与功能说明

### 3.1 页面结构

```
├── 登录/注册页面
├── 首页（排班总览）
├── 资源管理
│   ├── 歌手/乐手列表
│   ├── 新增/编辑歌手（含结构化档期配置）
├── 酒吧管理
│   ├── 酒吧列表
│   ├── 酒吧详情（子导航：排班表 | 歌手池 | 酒吧设置）
├── 排班工作台
│   ├── 选择酒吧和周期
│   ├── 自动生成排班
│   ├── 手动调整排班
│   ├── 顶班匹配
├── 记账结算
│   ├── 酒吧结算列表
│   ├── 按酒吧查看结算明细
├── 历史排班
```

### 3.2 登录/注册页面

#### 3.2.1 注册页面
- 用户输入手机号、验证码、密码完成注册
- 注册成功后跳转至登录页面

#### 3.2.2 登录页面
- 用户输入手机号、验证码完成登录
- 登录成功后进入首页

### 3.3 首页（排班总览）

- 展示各酒吧近期排班概况
- 提供快速入口：酒吧管理、资源管理、排班工作台

### 3.4 资源管理

#### 3.4.1 歌手/乐手列表（移动端优化）
- 以卡片形式展示艺人信息，每张卡片包含：姓名、类型标签、联系方式、风格标签、所属酒吧
- 卡片高度适中，信息层级清晰，字体大小：姓名text-lg，其他信息text-sm
- 支持按类型、风格筛选（筛选按钮 ≥48×48px）
- 支持新增艺人、编辑艺人、删除艺人（操作按钮 ≥48×48px）

#### 3.4.2 新增/编辑歌手页面（移动端优化）

**基础信息**
- 填写姓名、联系方式（输入框高度 ≥48px）
- 选择类型（歌手/乐手）
- 选择所属合作酒吧（多选，勾选后自动同步至对应酒吧歌手池）
- 配置擅长演唱风格标签（用户自行配置多个标签，如流行/R&B/爵士/摇滚等）
- 若为乐手，可选择固定分配的酒吧（单选）

**结构化档期配置（移动端优化）**
- 分为「长期固定档期」和「临时特殊档期」两类，临时优先级更高
- 长期固定档期：以卡片列表形式展示，每张卡片包含星期、可排时段、不可排时段、备注
- 临时特殊档期：以卡片列表形式展示，每张卡片包含具体日期、是否可排、可排时段、不可排时段、备注
- 支持手动逐条录入、修改、删除（操作按钮 ≥48×48px）
- 支持外部Excel/CSV一键导入，导入后自动覆盖更新档期数据
- 档期卡片示例：
  - 卡片标题：周一 / 2026-06-13
  - 卡片内容：可排时段 20:00–23:00 / 不可排时段 全天
  - 备注：请假
- 保存后返回艺人列表

### 3.5 酒吧管理

#### 3.5.1 酒吧列表（移动端优化）
- 以卡片形式展示酒吧信息，每张卡片包含：酒吧名称、地址、联系方式
- 卡片高度适中，信息层级清晰，字体大小：酒吧名称text-lg，其他信息text-sm
- 点击卡片进入酒吧详情

#### 3.5.2 酒吧详情（子导航）

**子导航一：排班表**
- 显示该酒吧历史各版本排班表（按月/周标记，例如「2026年5月第1周」）
- 以列表形式展示，每条记录包含版本名称、生成时间
- 点击某版本查看详情

**子导航二：歌手池（移动端优化）**
- 以卡片形式展示歌手池成员，每张卡片包含：姓名、风格标签、本酒吧专属单价
- 可在歌手池内为每位歌手设置本酒吧专属单价（未设则使用酒吧统一价）
- 支持从歌手池中移除歌手（但不删除歌手本身）
- 支持新增歌手（从全局艺人库选择加入）
- 操作按钮 ≥48×48px

**子导航三：酒吧设置（移动端优化）**
- 填写酒吧名称、地址、联系方式（输入框高度 ≥48px）
- 选择排班周期（一周一排/一月一排）
- 设置每晚演出总节数（数字输入，如3节或4节）
- 每节配置：
  - 节次名称（如第一节、第二节，支持自定义时间段如20:00-21:00）
  - 该节所需歌手人数（数字，各节独立配置，如第1节1人、第2节2人、第3节2人）
  - 该节所需歌手风格（复选框，从歌手池内现有风格标签中多选）
- 设置酒吧统一演出单价（元/场）
- 选择歌手池类型：
  - 封闭型：顶班只能从本店歌手池内匹配，不展示外部歌手
  - 开放型：顶班先显示本店歌手，再显示外部有档期歌手作为备选
- 保存按钮 ≥48×48px，保存后返回酒吧列表

### 3.6 排班工作台（移动端重设计）

#### 3.6.1 自动生成排班
- 选择酒吧（下拉选择框高度 ≥48px）
- 选择排班周期（本周/本月，或指定周/月）
- 点击「自动生成排班」按钮（按钮高度 ≥48px）
- 系统根据以下规则自动填充：
  1. 仅从本酒吧歌手池内选择歌手
  2. 歌手该时段有档期（长期固定档期+临时档期合并判断，临时优先）
  3. 风格匹配该节演出风格需求
  4. 均衡分配：当前排班周期内各歌手场次尽量均衡，避免某人被排过多
- 自动生成完成后，显示匹配反馈信息：「已成功匹配X个节次，Y个节次无可用歌手」
- 同一周期排班可覆盖更新（同酒吧同周期只保留一份当前版本）
- 新周期开始时另存新版本，历史版本保留

#### 3.6.2 手动调整排班（移动端重设计）
- 排班表以卡片式布局展示，按天组织，每天一个卡片
- 每张日期卡片包含：
  - 卡片标题：日期（如2026-05-26 周一），字体text-lg
  - 卡片内容：各节次的歌手分配，每节次一行
  - 每节次行包含：节次名称（如第一节 20:00-21:00）、歌手姓名、风格标签
  - 信息层级清晰：日期 → 节次 → 歌手
- 支持点击日期标题切换日期，或左右滑动切换日期
- 点击某节次的歌手姓名，弹出下拉选择框，显示该时段有档期、且在本酒吧歌手池内的可选歌手
- 下拉选择框中，每位歌手显示：姓名、风格标签、当期已排场次
- 不满足条件的歌手置灰显示（显示但不可选）
- 支持应急兜底：手动输入池外临时歌手姓名（不在系统歌手库中），记录到该排班中
- 保存按钮 ≥48×48px，保存排班结果

#### 3.6.3 顶班匹配（移动端优化）
- 选择需顶班的酒吧、日期、节次、当前缺席歌手（选择框高度 ≥48px）
- 系统显示匹配结果，分两区展示：
  - 本店可选：本酒吧歌手池内、该时段有档期的歌手
  - 外部可选（仅开放型酒吧显示）：池外、该时段有档期的歌手
- 每条结果以卡片形式展示，包含：姓名、联系方式、风格标签、当期已排场次
- 卡片高度适中，信息层级清晰，字体大小：姓名text-lg，其他信息text-sm
- 点击选择顶班人员后，更新排班表（选择按钮 ≥48×48px）

#### 3.6.4 排班导出
- 支持将当前排班表导出为PDF或Excel格式（导出按钮 ≥48×48px）

### 3.7 记账结算

#### 3.7.1 酒吧结算列表（移动端优化）
- 以卡片形式展示所有酒吧，每张卡片包含酒吧名称
- 点击卡片查看该酒吧结算明细

#### 3.7.2 结算明细（移动端优化）
- 选择历史排班版本（周期）后，系统自动生成结算表格
- 以卡片列表形式展示，每张卡片包含：歌手姓名、单场单价、演出总场次、结算总价、备注
- 卡片高度适中，信息层级清晰，字体大小：歌手姓名text-lg，其他信息text-sm
- 数据联动：
  - 歌手池内排班：自动取该歌手在本酒吧的专属价（无则取酒吧统一价），乘以场次
  - 手动录入的临时歌手：单价需手动填写，总价自动计算
- 支持备注说明特殊情况

### 3.8 历史排班（移动端优化）

- 以卡片列表形式展示所有酒吧的历史排班版本，每张卡片包含：酒吧名、周期名称、生成时间
- 卡片高度适中，信息层级清晰，字体大小：酒吧名text-lg，其他信息text-sm
- 点击卡片查看详情（只读）

## 4. 业务规则与逻辑

### 4.1 档期判断逻辑
- 系统判断某歌手在某具体日期/时间段是否可排时：
  1. 先查临时特殊档期是否有该日期的记录，有则以临时档期为准
  2. 无临时档期，则按长期固定档期中对应星期的规则判断
  3. 判断时段重叠：若酒吧该节演出时间段与歌手可排时段有重叠，则视为可用

### 4.2 均衡分配算法
- 统计当前排班周期内每位歌手已被分配的场次
- 优先选择已分配场次最少的歌手（在满足档期+风格条件的前提下）

### 4.3 版本管理
- 排班版本标识：酒吧ID+年份+周次/月份
- 同版本覆盖更新，跨版本另存
- 历史版本只读，不可修改

### 4.4 数据强关联
- 歌手档期修改 → 排班工作台实时更新置灰状态
- 歌手所属酒吧修改 → 对应酒吧歌手池自动同步
- Excel导入档期后 → 自动刷新所有关联排班校验

### 4.5 数据存储
- 所有数据（酒吧、艺人、排班、档期、结算）存储在Supabase
- 用户登录后可访问和管理所有数据

### 4.6 移动端设计规范
- 所有页面以375px宽屏为基准设计
- 触摸目标 ≥48×48px
- 信息层级清晰，使用卡片式布局替代表格
- 字体大小适配手机：标题text-lg，正文text-sm
- 按钮要大、间距要足
- 颜色对比度足够，深色模式也要清晰

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 歌手未提交档期 | 自动排班时不选该歌手；手动选择时置灰提示 |
| 无可用歌手 | 提示当前无可用歌手，建议手动输入临时歌手 |
| 顶班无内部匹配 | 开放型酒吧展示外部选项；封闭型提示无可用人员 |
| 删除酒吧时已有排班数据 | 提示是否确认删除，删除后相关排班数据一并处理 |
| 删除艺人时已有排班数据 | 提示是否确认删除，删除后相关排班数据一并处理 |
| 用户未登录 | 跳转至登录页面 |

## 6. 验收标准

1. 中介在移动端注册并登录系统
2. 新增一个酒吧，配置排班周期为一周一排，每晚3节，每节需要1名歌手，设置酒吧统一单价，选择歌手池类型为开放型
3. 新增2名歌手，分别配置长期固定档期和临时特殊档期，勾选所属酒吧后自动同步至酒吧歌手池
4. 进入排班工作台，选择该酒吧和本周，点击自动生成排班，系统根据档期和均衡分配规则自动填充排班表，并显示匹配反馈信息
5. 在移动端查看排班表，验证卡片式布局清晰，日期、节次、歌手信息层级分明，支持点击日期切换或左右滑动切换
6. 手动调整某天某节的歌手，点击歌手姓名弹出下拉选择框，选择其他可用歌手，保存排班
7. 模拟某歌手请假，使用顶班匹配功能，查看本店可选和外部可选区域的匹配结果（卡片形式），选择顶班人员并更新排班表
8. 导出当前排班表为PDF或Excel格式
9. 进入记账结算模块，选择该酒吧和排班版本，查看自动生成的结算卡片列表，验证单价和总价计算正确
10. 在历史排班中查看该酒吧的历史版本卡片列表，点击查看详情（只读）
11. 验证所有页面触摸目标 ≥48×48px，字体大小适配手机，信息层级清晰，颜色对比度足够

- 历史排班数据统计与分析
- 多用户权限管理（仅支持单一中介登录）
- 艺人端独立登录与档期自助提交
